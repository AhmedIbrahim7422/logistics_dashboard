NexusTrack Documentation

Premium Logistics & Fleet Management Dashboard Template (v1.0.0)

1. Introduction

NexusTrack is a modern, responsive, and multi-language logistics dashboard template built for speed and visual excellence. It features a complete RTL support system, interactive charts, and a clean code structure.

3. Customization

Translations

To add more languages or edit existing text, modify the assets/js/translations.js file. Add a new key to the translations object and update the data-i18n attributes in your HTML.

Styling

We use Tailwind CSS via CDN for rapid development. For production use, we recommend installing Tailwind via NPM and building the assets locally for smaller file sizes.

4. Backend Integration (API Data)

This template is completely backend-ready. All UI components (Charts, KPI Cards, Tables, Live Map tracking, and Fleet vehicle lists) are dynamically generated via JavaScript instead of hardcoded HTML.

How it works

We extract dummy data by using a centralized assets/js/data.js file containing a MockAPI object. The script.js file then listens to page load events, requests this data asynchronously, and writes standard HTML dynamically inside empty container ids.

Connecting to your own database

To hook NexusTrack up to a real backend (Node.js, Laravel, Django, etc.), simply modify the functions in data.js to return your actual REST API calls via fetch or axios. 

// Inside assets/js/data.js
const MockAPI = {
    getDashboardKPIs: async () => {
        // Replace mock with:
        // const response = await fetch('/api/v1/kpis');
        // return await response.json();
    }
}

Note: Ensure your API response JSON mirrors the structure of the mock data provided within data.js so that the DOM rendering algorithms in script.js continue functioning identically.
